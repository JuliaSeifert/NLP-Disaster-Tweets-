Description of Problem and Data

The overall goal of this kaggle competition is to be able to predict whether a tweet in this dataset is about a real disaster or not. 1=yes, real disaster. 0=no, not about a real disaster. The test data contains 4 columns/variable that are id, text, location, and keyword. The train set has these same 4 columns but an additional one of target. The id depicts the ID number for the tweet, the text is the actual text the tweet contains, location is the location where the tweet was sent, keyword is a keyword from the tweet, and target is a 1 or a 0 denoting if a tweet is real or not.

Model Architecture

The word embeding process I am going to use is TF-IDF. TF-IDF stands for Term Frequency-Inverse Document Frequency and it is an algorithm used to determine how relevant words in a document are by using the frequency of the word. I thought this method of word embeding would be the best over other methods because it takes less steps to get the relevancy of the terms from this data set into a vector form, other methods would take extra steps and time to do that.

TF-IDF

LSTM Model

I chose to use a LSTM neural network model because it will be able to solve the vanishing gradient problem that often arises with simpler RNNs. This is because LSTM models allow gradient to flow unchanged which I think will benefit this dataset more. I just have to be careful about overfitting because LSTM models tend to do that more than other neural network models. To combat overfitting I am going to adjust and add hyperparameters of dropout and regularizers.

Conclusion

In the future I would fine tune the hyperparameters even more to see which would create the best model accuracy. There are also other neural network models that I would run to see if they performed better at classification for this data set.

Hyperparameter tuning did not improve these models. I think the only thing that would improve prediction is some more data cleaning, looking at collinearities of features and using a different deep learning or neural network model to use for prediction on this data set.

Takeaways and learnings from this mini project shows that it is difficult to chose a correct neural network model for classification and prediction. Maybe a feedforward neural network would perform better on this data set because it is text data and feedforward neural networks perform well on text prediction data. Feedforward models are also flexible so it could scale well to this dataset.
